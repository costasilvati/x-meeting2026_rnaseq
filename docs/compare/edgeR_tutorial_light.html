PASSO 01

A Matriz de Counts

O edgeR trabalha com contagens brutas de reads mapeados — uma tabela genes × amostras com valores inteiros não normalizados.

📊 Dataset de referência — 6 genes × 6 amostras

🧬 Por que counts brutos?
O modelo Binomial Negativa do edgeR precisa da natureza discreta dos dados. FPKM/TPM destroem essa propriedade estatística.
⚠️ Replicatas biológicas
Mínimo recomendado: n=3 por grupo. Sem replicatas, a variabilidade biológica não pode ser estimada de forma confiável.

PASSO 02

Criando o DGEList

O DGEList é o objeto central do edgeR. O primeiro cálculo automático é o tamanho de cada biblioteca (lib.size = soma de todos os counts por amostra).

📐 lib.size — Tamanho da Biblioteca
lib.sizej = Σi countsij
Soma de todos os reads mapeados na amostra j
CTRL_1120 + 800 + 50 + 300 + 20 + 1000= 2.290
CTRL_2135 + 820 + 45 + 290 + 18 + 980= 2.288
CTRL_3110 + 790 + 55 + 310 + 22 + 1020= 2.307
TREAT_1540 + 810 + 12 + 305 + 95 + 990= 2.752
TREAT_2580 + 795 + 10 + 295 + 100 + 1010= 2.790
TREAT_3510 + 825 + 14 + 315 + 88 + 1000= 2.752
💻 Código R
dge <- DGEList(counts=counts, group=group)
dge$samples  # lib.size calculado automaticamente
#         group lib.size norm.factors
# CTRL_1   CTRL     2290            1
# TREAT_1 TREAT     2752            1

keep <- filterByExpr(dge)
dge <- dge[keep, , keep.lib.sizes=FALSE]
PASSO 03

Normalização TMM

Trimmed Mean of M-values — corrige diferenças de composição entre bibliotecas, não apenas de tamanho total.

📐 M-values e A-values
Mg = log₂(Ygk/Nk) − log₂(Ygr/Nr)
Ag = ½·[log₂(Ygk/Nk) + log₂(Ygr/Nr)]
Y=count, N=lib.size, k=amostra teste, r=referência (CTRL_2)

Depois de calcular M e A para cada gene, o método trima 30% dos extremos de M e 5% de A, calcula a média ponderada dos M restantes e eleva a 2:

norm.factor = 2média ponderada (M não trimados)
🔢 Cálculo para TREAT_1 vs CTRL_2 (referência)
GeneAlog₂(540/2752) − log₂(135/2288) = −2.349 − (−4.083)M = +1.734
GeneBlog₂(810/2752) − log₂(820/2288)M = −0.285
GeneDlog₂(305/2752) − log₂(290/2288)M = −0.193
GeneFlog₂(990/2752) − log₂(980/2288)M = −0.252

GeneA (M alto) é trimado. Média ponderada dos restantes ≈ −0.243

norm.factor(TREAT_1) = 2−0.243 ≈ 0.846

PASSO 04

Estimando a Dispersão

A dispersão mede a variabilidade biológica além do esperado por Poisson. É o parâmetro mais crítico do modelo Binomial Negativa.

📐 Variância da Binomial Negativa
Var(Y) = μ + φ · μ²
μ = média esperada · φ = dispersão · BCV = √φ

Com n=3, as estimativas individuais por gene são instáveis. O edgeR aplica shrinkage bayesiano, combinando dispersão por gene (tagwise) com a tendência global (trended).

📋 Dispersões estimadas
Geneφ tagwiseBCV = √φInterpretação

GeneB0.0089.0%Gene constitutivo estável
GeneC0.04521.2%Mais variável (counts baixos)
GeneD0.01010.0%Comportamento estável
GeneE0.03518.7%Variável (counts baixos)
GeneF0.0078.4%Gene housekeeping muito estável
💻 Código R
dge <- estimateDisp(dge)
sqrt(dge$common.dispersion)  # BCV comum ≈ 0.10
plotBCV(dge)
PASSO 05

Modelo GLM-QLF

O GLM estima o efeito do tratamento (= log₂FC) para cada gene. O QLF (Quasi-Likelihood F-test) produz p-valores mais conservadores e robustos para amostras pequenas.

📐 Preditor linear do GLM
log(μij) = log(Nj) + β₀i + β₁i · xj
N_j = eff.lib.size · β₀ = expressão basal (CTRL) · β₁ = efeito TREAT = log₂FC
🔢 log₂FC calculado para cada gene
GeneAlog₂(média TREAT / média CTRL) = log₂(543.3 / 121.7)+2.16
GeneBlog₂(810 / 803.3)+0.02
GeneClog₂(12 / 50)−2.06
GeneDlog₂(305 / 300)+0.02
GeneElog₂(94.3 / 20)+2.24
GeneFlog₂(1000 / 1000)0.00
💻 Código R
design <- model.matrix(~group)
fit    <- glmQLFit(dge, design)
qlf    <- glmQLFTest(fit, coef=2)
PASSO 06

Resultados & FDR

O edgeR aplica o método Benjamini-Hochberg para controlar a taxa de falsos positivos (FDR). O resultado final é uma tabela com log₂FC, p-valor e FDR.

📐 Correção BH para múltiplos testes
FDR(i) = p(i) × m / i
p_(i) = p-valor de rank i · m = total de genes testados
