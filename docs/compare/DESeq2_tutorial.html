Dados & DESeqDataSet

🆚 DESeq2 vs edgeR — Ponto de partida idêntico

Ambos recebem a mesma matriz de counts inteiros. A diferença começa na estratégia de normalização (Size Factors vs TMM) e no método de estimativa de dispersão.

💻 Código R
library(DESeq2)

# Metadados das amostras
coldata <- data.frame(
  condition = factor(c("CTRL","CTRL","CTRL","TREAT","TREAT","TREAT")),
  row.names = colnames(counts)
)

# Criar o objeto DESeqDataSet
dds <- DESeqDataSetFromMatrix(
  countData = counts,
  colData   = coldata,
  design    = ~condition
)

# Filtrar genes com poucos counts
dds <- dds[rowSums(counts(dds)) >= 10, ]
PASSO 02

Normalização por Size Factors

O método de Anders & Huber (2010) usa a mediana das razões de cada gene em relação a uma amostra de referência geométrica — robusto a genes com expressão muito alta.

📐 Algoritmo de Size Factors — 3 etapas

Etapa 1: Calcular a média geométrica de cada gene através de todas as amostras:

gi = (∏j countsij)1/n = exp(mean(log(countsij)))
g_i = pseudo-referência do gene i

Etapa 2: Para cada amostra, calcular a razão de cada gene em relação à sua pseudo-referência:

rij = countsij / gi
razão do gene i na amostra j

Etapa 3: O size factor de cada amostra é a mediana dessas razões:

sj = medianai(rij)
Mediana torna o método robusto a genes diferencialmente expressos

💡 Por que size factors ≈ 1.0 mesmo com bibliotecas diferentes?
A mediana ignora GeneA (UP) e GeneC (DOWN). Os genes "estáveis" (B, D, F) dominam o cálculo — exatamente o comportamento desejado. O TMM do edgeR chega a um efeito similar, mas pela via das M-values.
💻 Código R
# DESeq2 calcula size factors automaticamente dentro de DESeq()
# Mas pode ser chamado separadamente:
dds <- estimateSizeFactors(dds)
sizeFactors(dds)

# Counts normalizados (para visualização — NÃO usados no modelo)
counts(dds, normalized=TRUE)
PASSO 03

Estimativa de Dispersão

O DESeq2 usa uma abordagem de 3 fases para estimar a dispersão: por gene, ajuste de curva (fitted) e shrinkage por MAP (Maximum A Posteriori).

📐 Variância da Binomial Negativa no DESeq2
Var(Kij) = μij + αi · μij²
α_i = dispersão do gene i (equivalente ao φ do edgeR)
🔢 As 3 fases da estimativa de dispersão DESeq2
Fase 1 — Dispersão por gene (MoM)

Método dos Momentos: iguala variância observada à variância da BN e resolve para α.

GeneBVar_obs=233 / μ=803, α_hat = (233−803)/803² α ≈ 0.001
GeneCVar_obs=52 / μ=27 (norm), α_hat = (52−27)/27²α ≈ 0.034
GeneFVar_obs=280 / μ=1000, α_hat = (280−1000)/1000²α ≈ 0.001
Fase 2 — Curva de dispersão fitted

Ajusta uma curva paramétrica: α̂_fitted(μ) = a₀/μ + a₁. Genes com expressão baixa têm dispersão maior — isso é esperado biologicamente e capturado pela curva.

Fase 3 — MAP Shrinkage

A dispersão final de cada gene é puxada em direção à curva fitted usando uma prior log-Normal. Com n pequeno, estimativas por gene são instáveis — o shrinkage estabiliza sem perder informação real de genes muito variáveis.

αfinal = argmax P(α | dados) × P(α | curva fitted)
MAP: combina verossimilhança dos dados com prior da curva
💻 Código R
dds <- estimateDispersions(dds)

# Visualizar as 3 camadas de dispersão
plotDispEsts(dds)
# Pontos pretos = por gene | curva vermelha = fitted | azul = MAP final

# Acessar dispersões
dispersions(dds)  # dispersões finais (MAP)
PASSO 04

Modelo Binomial Negativa

Com size factors e dispersões estimados, o DESeq2 ajusta um GLM-BN para cada gene usando máxima verossimilhança.

📐 O modelo de contagens do DESeq2
Kij ~ NegBin(μij, αi)
μij = sj · qij
K_ij = count observado · s_j = size factor · q_ij = expressão normalizada
log(qij) = βi0 + βi1 · xj
β_i0 = intercepto (nível basal CTRL) · β_i1 = efeito TREAT = LFC
🔢 LFC estimado para cada gene
GeneAlog₂(média TREAT norm / média CTRL norm) ≈ log₂(543/121.7)LFC = +2.16
GeneBlog₂(810 / 803)LFC = +0.01
GeneClog₂(12 / 50)LFC = −2.06
GeneDlog₂(305 / 300)LFC = +0.02
GeneElog₂(94.3 / 20)LFC = +2.24
GeneFlog₂(1000 / 1000)LFC = 0.00
PASSO 05

Teste de Wald

O DESeq2 testa H₀: β₁ = 0 usando a estatística de Wald — razão entre o coeficiente estimado e seu erro padrão. Mais rápido que o LRT e adequado para comparações simples.

📐 Estatística de Wald
W = β̂₁ / SE(β̂₁)
Sob H₀, W ~ Normal(0,1) → p-valor da distribuição normal padrão
🔢 Cálculo para genes selecionados
GeneAβ̂₁ = 2.16 · log(2) = 1.497; SE ≈ 0.088W = 17.01 → p < 0.0001
GeneCβ̂₁ = −2.06 · log(2) = −1.428; SE ≈ 0.142W = −10.06 → p < 0.0001
GeneBβ̂₁ = 0.01 · log(2) = 0.007; SE ≈ 0.034W = 0.21 → p = 0.834
⚠️ Wald vs LRT
O teste de Likelihood Ratio (LRT) é preferido quando há múltiplos níveis de um fator ou quando genes têm dispersão muito alta. Para comparações simples 2 grupos, o Wald é adequado e mais rápido.
💻 Código R
# nbinomWaldTest é chamado automaticamente por DESeq()
dds <- nbinomWaldTest(dds)

# Extrair resultados (TREAT vs CTRL)
res <- results(dds,
         contrast = c("condition", "TREAT", "CTRL"),
         alpha    = 0.05)
summary(res)
PASSO 06

LFC Shrinkage

Funcionalidade exclusiva do DESeq2: encolhimento do log₂FC estimado para genes com poucos counts ou alta variabilidade — tornando os efeitos mais confiáveis e comparáveis.

📐 O problema sem shrinkage

Genes com counts baixos tendem a ter LFCs inflados: se um gene tem 2 reads no CTRL e 8 no TREAT, o LFC = 2 parece enorme — mas é baseado em muito pouca informação.

Métodos de shrinkage disponíveis
apeglmPrior de Cauchy. Shrinkage adaptativo — genes com forte evidência são pouco afetados⭐ Recomendado
ashrAdaptive shrinkage — estima a distribuição dos LFCs a partir dos dadosAlternativa
normalPrior Normal. Método original do DESeq2 v1 — menos recomendado hojeLegado

GeneE+2.24+2.10Leve (counts moderados)

GeneD+0.02+0.004Forte (LFC próximo de zero)

💻 Código R
library(apeglm)
res_lfc <- lfcShrink(dds,
             coef = "condition_TREAT_vs_CTRL",
             type = "apeglm")

# Comparar LFCs antes e depois
plot(res$log2FoldChange, res_lfc$log2FoldChange,
     xlab="LFC original", ylab="LFC após shrinkage")
abline(0, 1, lty=2)

# Pipeline completo em uma linha:
dds <- DESeq(dds)
# equivale a: estimateSizeFactors → estimateDispersions → nbinomWaldTest

res     <- results(dds, contrast=c("condition","TREAT","CTRL"), alpha=0.05)
res_lfc <- lfcShrink(dds, coef="condition_TREAT_vs_CTRL", type="apeglm")

# MA plot
plotMA(res_lfc, ylim=c(-4,4))

# Genes significativos
sig <- subset(res_lfc, padj < 0.05 & abs(log2FoldChange) > 1)
